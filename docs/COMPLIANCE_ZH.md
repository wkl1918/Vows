# 合规与许可证说明（重点）

本文件用于开源发布前自查，不构成法律意见。

## 1. IndexTTS2 使用约束（你当前项目相关）
你集成的 IndexTTS2 许可证并非宽松 MIT/Apache，需要遵守其附加条款：
- 保留原版权与许可文本
- 分发衍生品时要明确“改动不代表原权利人背书”
- 特定规模商业主体可能需要另行商业许可
- 不得用于违法违规或高风险场景

建议在项目 README 与发布页写明：
- “本项目集成 IndexTTS2，遵循其原许可条款，详见官方仓库 License。”

## 2. 语音克隆素材合规
- 仅使用你有授权的音频做克隆参考
- 不要公开上传政治人物/公众人物克隆样本
- 不要上传未经授权的影视、演讲、课程素材

## 3. 第三方模型与缓存
本仓库不再分发以下内容：
- `checkpoints/`、`hf_cache/`、`*.pth`、`*.pt`、`*.safetensors`
原因：
- 文件体积大
- 第三方许可证可能不允许二次再分发

## 4. 密钥与隐私
严禁上传：
- `.env`
- 任意 token/API key
- 包含个人信息的日志与样本数据

## 5. 你发布前的最终清单
1. 全仓库检索 `sk-`、`hf_`、`token`，确认没有真实密钥
2. 确认 `.gitignore` 已生效
3. 确认无模型权重、无缓存、无运行产物
4. README 写明依赖模型需用户自行下载
5. 保留/附带上游许可证文件与引用信息
